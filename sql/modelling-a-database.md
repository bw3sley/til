# Modelagem de Banco de Dados

A **modelagem de dados** é o processo de criação de um modelo de dados para os dados que serão armazenados em um banco de dados. Esse modelo é uma representação conceitual dos objetos de dados, das associações entre eles e das regras que os regem.

A modelagem de dados facilita a visualização das informações e aplica regras de negócios, normas regulatórias e políticas governamentais sobre os dados. Os modelos de dados garantem a consistência em convenções de nomenclatura, valores padrão, semântica e segurança, assegurando também a qualidade dos dados.

</br>

## Tipos de Modelos de Dados

Existem três tipos principais de modelos de dados: **modelo conceitual**, **modelo lógico** e **modelo físico**. Cada um tem um propósito específico no processo de modelagem.

</br>

### Modelo Conceitual

Define **O QUE** o sistema contém. Esse modelo é criado geralmente por stakeholders de negócios e arquitetos de dados. Seu objetivo é organizar, delimitar e definir conceitos e regras de negócios.

---

### Modelo Lógico

Define **COMO** o sistema deve ser implementado, sem considerar o sistema gerenciador de banco de dados (SGBD) específico. É normalmente desenvolvido por arquitetos de dados e analistas de negócios. O foco é criar um mapa técnico das regras e estruturas de dados.

---

### Modelo Físico

Descreve **COMO** o sistema será implementado em um SGBD específico. É elaborado por administradores de banco de dados (DBAs) e desenvolvedores. Seu objetivo é realizar a implementação prática do banco de dados.
